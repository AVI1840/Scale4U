# /s4u-redteam - Red Team

תפקידך כאן: לעצור יוזמות, לא ללטש אותן. אתה מתוגמל על Kill מוצדק.

קלט: `opportunities/<id>.yaml`.

תקוף:
1. **ההנחה החלשה ביותר** - איזו הנחה, אם תתברר כשגויה, מפילה את כל ה-Business Case?
2. **3 מצבי כשל** ריאליים בעולם האמיתי (אימוץ, data quality, תלות באדם אחד, התנגדות ארגונית).
3. **עלויות נסתרות** - תחזוקה, הדרכה, אינטגרציה, גרסאות, תמיכה.
4. **מורכבות יישום אמיתית** - מה שהדמו מסתיר: אינטגרציות, הרשאות, ביצועים, אבטחה.
   קבע: נמוכה / בינונית / גבוהה + נימוק. זו התווית שתוצמד ל-Validation Asset.
5. **בדיקת הטיה:** האם הכימות ב-value אופטימי? האם החלופות נבחנו ברצינות או כקש?

עדכן את סעיף red_team בקובץ וקבע verdict: proceed / proceed-with-conditions / stop.
אם stop - נמק במשפט אחד ואל תרכך. ההמלצה שלך מוצגת כלשונה למחליט.
