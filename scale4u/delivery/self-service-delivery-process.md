# תהליך פיתוח מהיר Self-Service - מקצה לקצה

## מתי מפעילים
יוזמה שנותבה ל-**Self-Service** (ציון סיכון 0-4, אפס נגיעה בדאטה אישי/זכאות/ליבה).
דוגמאות: דשבורד, מחשבון, עוזר פנימי, כלי ניהול ידע, אוטומציה פנימית.

---

## השלבים (יום-יומיים מניתוב ליישום)

### שלב 1: הגדרת Scope (שעה)
**מי:** מפעיל Scale4U / רפרנט AI
**כלי:** Claude Code

```
/s4u-asset [id]
```

**Output:**
- מה בונים (מסך אחד, פיצ'ר אחד)
- מי המשתמש (5-8 אנשים ספציפיים לבדיקה)
- מה קריטריון ההצלחה ("5/8 אומרים שישתמשו")
- מה OUT of scope

**כלל:** אם לא אפשר לתאר ב-3 משפטים - זה לא Self-Service.

---

### שלב 2: פיתוח (2-4 שעות)
**מי:** רפרנט AI / מפעיל
**כלי:** Claude Code

**הפקודה:**
```
בנה [דשבורד/מחשבון/עוזר] שעושה [X].
משתמש: [תפקיד].
קלט: [מה המשתמש מזין].
פלט: [מה הוא מקבל].
עיצוב: RTL, עברית, Heebo, light theme, standalone HTML.
```

**Output:** קובץ HTML אחד עצמאי שעובד offline.

**כללים:**
- HTML standalone - אין CDN, אין שרת, אין DB
- RTL + עברית + Heebo font
- נגישות בסיסית (contrast, font-size, labels)
- דאטה סינתטי בלבד (אם צריך דוגמאות)
- מקסימום 4 שעות פיתוח. אם יותר - זה לא Self-Service.

---

### שלב 3: בדיקה מול משתמשים (יום)
**מי:** רפרנט AI + 5-8 משתמשי קצה
**כלי:** פתיחת HTML בדפדפן + 5 דקות שיחה

**שאלות:**
1. "תשתמש בזה?" (כן/לא/בתנאי ש...)
2. "מה חסר?" (דבר אחד)
3. "מה מבלבל?" (דבר אחד)

**קריטריון:** 5/8 אומרים "כן" ללא הסתייגות מהותית → ממשיכים.
אם לא → תיקון של שעה ובדיקה חוזרת.

---

### שלב 4: פריסה ורישום (30 דקות)
**מי:** מפעיל Scale4U
**כלי:** GitHub Pages / SharePoint / תיקייה משותפת

**פעולות:**
1. העלאת הקובץ לאתר Scale4U או תיקייה משותפת
2. רישום ב-`assets/fast-lane-registry.yaml`:
   ```yaml
   - id: [id]
     name: "[שם הכלי]"
     owner: "[שם הרפרנט]"
     deployed: "YYYY-MM-DD"
     type: "[dashboard/calculator/assistant/automation]"
     users: "[מי משתמש]"
     next_review: "YYYY-MM-DD"  # +90 ימים
   ```
3. שליחת הודעה לבעלים העסקי: "הכלי מוכן. הנה הקישור."

---

### שלב 5: ביקורת רבעונית (15 דקות)
**מתי:** כל 90 יום
**מי:** מפעיל Scale4U

**בדיקה:**
- [ ] עדיין בשימוש?
- [ ] צריך עדכון?
- [ ] צצו סיכונים חדשים (דאטה אישי שנכנס, scope שנפח)?
- [ ] צריך לשדרג ל-Fast Pilot או Enterprise?

---

## Checklist - Self-Service Delivery

```
□ Scope מוגדר (3 משפטים)
□ משתמשים מזוהים (5-8 שמות)
□ HTML נבנה ונבדק
□ 5/8 משתמשים אישרו
□ נפרס (URL/קובץ)
□ נרשם ב-registry
□ בעלים עודכן
□ ביקורת רבעונית בלוח
```

## דוגמה: "דשבורד מעקב ביצועי אגף"

| שלב | זמן | Output |
|---|---|---|
| Scope | 30 דק' | "דשבורד שמציג 15 KPIs של אגף תכנון, מעודכן ידנית" |
| פיתוח | 3 שעות | HTML file עם טבלה + גרפים CSS |
| בדיקה | יום | 6/8 מנהלי יחידות אישרו |
| פריסה | 20 דק' | עלה לאתר + נרשם ב-registry |
| **סה"כ** | **~2 ימי עבודה** | **כלי חי בשימוש** |
