# /s4u-value - כימות וחלופות

קלט: `opportunities/<id>.yaml` (אם לא צוין id - שאל).

תהליך:
1. **כימות הסטטוס-קוו:** בנה את חשבון העלות מהנתונים שבקובץ ומשאלות למשתמש.
   נוסחה שקופה (למשל: מקרים בחודש x דקות למקרה x עלות שעה). כל רכיב חסר = TBV + מי מספק.
2. **השערת ערך:** נסח hypothesis מדיד + 2-3 מדדים עם baseline/target. baseline חסר = TBV עם תכנית השגה ביום 4.
3. **חלופות:** מלא assessment כן ל-3 החלופות לפחות, כולל "לא לעשות" ופתרון לא-טכנולוגי.
   אם חלופה לא-טכנולוגית פותרת 80% מהכאב - אמור זאת במפורש. זו הצלחת המערכת, לא כישלון.
4. עדכן את הקובץ. הרץ אוטומטית את לוגיקת /s4u-redteam (או הפנה אליה) - G1 לא עובר בלעדיה.
5. הצג המלצת G1 לפי `gates/gates.md` עם צ'קליסט מסומן.

פלט: קובץ מעודכן + המלצת G1. אל תייצר Business Case נפרד - הקובץ הוא ה-Business Case.
